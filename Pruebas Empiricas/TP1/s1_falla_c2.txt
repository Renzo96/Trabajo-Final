**2. ¿Qué significa que un archivo esté "Preparado" (Staged)?**

Que un archivo esté Staged significa que ya fue marcado con `git add` para ser incluido en el próximo commit, pero ese commit todavía no se hizo. Es el paso intermedio entre modificar un archivo y confirmarlo definitivamente en el historial con `git commit`.

**3. ¿Para qué sirve `.gitignore`?**

El archivo `.gitignore` le dice a Git qué archivos o carpetas debe ignorar y nunca rastrear. Un ejemplo típico es un archivo `.env` con contraseñas o claves de API, que nunca debería subirse a un repositorio público.

**4. ¿Cómo ayudan las "Ramas" (Branches)?**

Las ramas permiten crear una copia paralela y aislada del código para probar algo nuevo sin afectar la rama `main`. Los cambios solo se integran a `main` cuando el programador decide hacer el merge.
