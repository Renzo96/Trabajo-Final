Caso A: Generación de función matemática compleja en Python

Para este caso, que requiere máxima precisión y determinismo, recomiendo:

- Temperatura: 0.1
- Top-p: 0.1

Justificación: Según la teoría de Predicción de Tokens, el modelo genera texto prediciendo el siguiente token más probable en cada paso. La temperatura controla la "suavidad" de la distribución de probabilidad: valores bajos (cercanos a 0) hacen que la distribución sea más "picuda", concentrando casi toda la probabilidad en los tokens más probables. Con temperatura 0.1, el modelo seleccionará casi siempre el token de mayor probabilidad, eliminando prácticamente la aleatoriedad. El Top-p (nucleus sampling) en 0.1 restringe aún más el pool de tokens candidatos: solo considera aquellos tokens cuya probabilidad acumulada alcance el 10%, lo que en la práctica deja únicamente los tokens más probables. Esta combinación garantiza un output determinista y preciso, esencial para código matemático donde un solo error de sintaxis o lógica invalida todo el programa.

Caso B: Escritura de post creativo para redes sociales

Para este caso, que requiere creatividad, variedad y un tono llamativo, recomiendo:

- Temperatura: 0.9
- Top-p: 0.95

Justificación: En la teoría de Predicción de Tokens, una temperatura alta (0.9) "aplana" la distribución de probabilidad, dándole más peso relativo a tokens que originalmente tenían baja probabilidad. Esto aumenta significativamente la aleatoriedad del sampling: el modelo ya no elige siempre el token más obvio, sino que explora alternativas menos predecibles, generando frases más originales y variadas. El Top-p de 0.95 amplía el pool de tokens candidatos: considera todos los tokens cuya probabilidad acumulada llegue al 95%, lo que incluye muchas opciones diversas. Esta combinación es ideal para un post de redes sociales sobre un lanzamiento de startup, donde se necesita un lenguaje fresco, creativo y no genérico que capte la atención del público.
